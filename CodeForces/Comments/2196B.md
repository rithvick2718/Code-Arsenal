Consider the following solution:

For each $i$, we will check all possible candidates $j$ that can form beautiful pairs. To do this, we will iterate through such $j=i+k⋅a_i$, where $k>0$.

This solution works slowly for small $a_i$. Let's try to speed it up using square root decomposition. We will introduce a parameter $B$ and will iterate through candidates for $i$ in different ways depending on whether $a_i<B$ or $a_i≥B$:

- If $ a_i≥B $, we will iterate through candidates in both directions from $i$ (not just forward), meaning for each such $i$, the set of candidates will be $i+k⋅a_i$, where $k$ is any integer;
- If $a_i<B$, we will also iterate through candidates only forward, but we will introduce an additional restriction on $k$, namely $k<B$. 

Let's estimate the maximum possible number of candidates for these two cases:

- In the first case, for all such $i$ where $a_i≥B$, it is $\frac{n}{B}$, since we move in steps of $B$;
- In the second case, it is clear that there are no more than $B$ candidates, as we have introduced such a restriction ourselves. 

If we choose $B=\left⌈ \sqrt{n} \right⌉$, then the number of candidates for each case is no more than $\left⌈ \sqrt{n} \right⌉$, so the entire algorithm solves the problem in $\mathcal{O}(n \sqrt{n})$.

Let's think about why such an algorithm will be correct. Consider the cases of a beautiful pair $(i,j)$, depending on the values of $a_i$ and $a_j$:

- If $a_i<B$ and $a_j<B$, then the pair will be counted when we iterate through candidates for $i$;
- If $a_i≥B$ and $a_j<B$, then the pair will be counted when we iterate through candidates for $j$;
- If $a_i<B$ and $a_j≥B$, then the pair will be counted when we iterate through candidates for $i$; 

Note that there is no case where $a_i≥B$ and $a_j≥B$, since we chose $B=\left⌈ \sqrt{n} \right⌉$, which means that in such a pair $a_i⋅a_j≥n$, and the maximum value of $j−i=n−1$.

Therefore, our algorithm is correct.
